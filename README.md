# トラップ・オセロ（プロトタイプ）

通常のオセロに**「伏せた罠」**と**「見切り」**を足した、対 CPU の1人用ゲーム。相手の着手そのものを利益に変える読み合いを楽しむ。

- **ビルド不要・外部依存ゼロ・オフライン完結**。データの保存も外部送信もしない。
- 構成は「エントリ HTML ＋ スタイル（`styles/`）＋ 責務別スクリプト（`src/`）」。ES Modules / `fetch` は使わず、classic `<script>` ＋ `window.TO` 名前空間で連携する。

## 遊び方

[`index.html`](index.html) をブラウザで直接開く（`file://`）か、任意の静的サーバーで配信する。

```
npx --yes http-server -p 8777 -c-1
```

あなたが黒（先手）、CPU が白。CPU の強さは **初級／中級（既定）／上級** から選べる（サイドパネルの「CPU の強さ」）。難易度を変えても進行中の対局はそのままで、次の「最初から」で反映される。

- **罠**：手札3個。自分の手番に、石を置く前に空きマスへ1個まで伏せられる。相手がそこへ石を置くと、その手を通常どおり解決したあと、**置かれた石と裏返った石を全部こちらが奪う**。
- 罠は**相手のものだけ**が自分に反応する。同じマスを両者が伏せても互いに残る。自分の罠を自分で踏むと無効化されるだけ。
- **見切り**：1試合1回。そのターンだけ罠を無効化する。踏んだ罠は解除・公開され、成果はそのまま手に入る。空振りしても消費し、**使用は相手に見える**。
- 双方に打つ手が無くなったら終局。石数の多い側の勝ち。

ルールの正確な定義は [`spec.md`](spec.md) §2 を参照。

## 構成

```
index.html      … マークアップと読み込み順の宣言のみ
styles/base.css … デザイントークン・レイアウト・サイドパネル
styles/board.css… 盤・石・罠・演出
src/config.js   … 定数と調整用パラメータ（バランス調整はここだけ）
src/rules.js    … オセロの純粋ルール
src/game.js     … 対局状態と状態遷移（罠・見切り）
src/cpu.js      … CPU のファサードと共通処理
src/cpu/*.js    … 難易度ごとの判断（beginner / intermediate / advanced）
src/view.js     … 描画・アニメーション
src/main.js     … 進行制御・入力・起動
```

依存は一方向（`config → rules → game → cpu (→ cpu/<難易度>) → view → main`。難易度は `cpu` の葉で、`view` / `main` からは見えない）。`rules` / `game` / `cpu` は DOM に触れず、難易度の実装は公開情報（`game.publicView()`）しか見ない。詳細は [`spec.md`](spec.md) §4。
